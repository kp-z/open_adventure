# Godot HTTP 请求并发问题修复

**创建日期**: 2026-03-18
**作者**: Claude Opus 4.6
**状态**: 已完成

## 问题描述

在 Godot 游戏中出现多个 HTTP 请求相关的错误：

### 错误 1: HTTPRequest 并发冲突
```
ERROR: HTTPRequest is processing a request. Wait for completion or cancel it before attempting a new one.
at: request_raw (scene/main/http_request.cpp:120)
```

**错误码**: 44 (ERR_BUSY)
**原因**: `MicroverseAPIClient` 只有一个 `HTTPRequest` 节点，当多个 API 调用同时发起时，会导致冲突。

### 错误 2: Meta 值访问错误
```
ERROR: The object does not have any 'meta' values with the key 'bound_agent_id'.
at: get_meta (core/object/object.cpp:1155)
```

**原因**: 使用了 `get_meta("bound_agent_id", null)` 的两参数形式，但 Godot 4 中 `get_meta` 只接受一个参数，需要先用 `has_meta` 检查。

### 错误 3: AgentDialogSystem 缺失
```
[CharacterManager] 错误: 找不到 AgentDialogSystem
```

**原因**: 场景中缺少 AgentDialogSystem 节点。

## 问题分析

### 1. HTTP 请求并发问题

**原始代码**（`microverse/script/api/MicroverseAPIClient.gd`）:
```gdscript
var http_request: HTTPRequest

func _ready():
    http_request = HTTPRequest.new()
    add_child(http_request)
    http_request.request_completed.connect(_on_request_completed)

func _make_request(method: String, endpoint: String, body: Dictionary = {}) -> Dictionary:
    # ...
    var error = http_request.request(url, headers, _get_http_method(method), request_body)
    # ...
```

**问题**:
- 只有一个 `HTTPRequest` 节点
- 当多个 API 调用同时发起时（例如：获取 Agent 列表 + 获取任务列表），第二个请求会失败
- HTTPRequest 在处理请求时不能接受新的请求

**影响**:
- Agent 绑定面板无法正常加载数据
- 任务列表获取失败
- 用户体验差

### 2. Meta 值访问问题

**原始代码**（`microverse/script/ui/AgentBindingPanel.gd`）:
```gdscript
var bound_agent_id = current_character.get_meta("bound_agent_id", null)
```

**问题**:
- Godot 4 中 `get_meta` 只接受一个参数
- 如果 meta 值不存在，会抛出错误
- 需要先用 `has_meta` 检查

## 解决方案

### 方案 1: HTTP 请求池（已实施）

创建一个 HTTPRequest 节点池，支持多个并发请求。

**实现思路**:
1. 创建多个 HTTPRequest 节点（默认 5 个）
2. 每次请求时，从池中获取空闲的节点
3. 如果所有节点都在忙，等待一段时间后重试
4. 请求完成后，节点自动变为空闲状态

**代码实现**:
```gdscript
# HTTP 请求节点池
var http_request_pool: Array = []
var max_concurrent_requests = 5

func _ready():
    # 创建 HTTPRequest 节点池
    for i in range(max_concurrent_requests):
        var http_request = HTTPRequest.new()
        add_child(http_request)
        http_request_pool.append(http_request)

func _get_available_http_request() -> HTTPRequest:
    """获取可用的 HTTPRequest 节点"""
    for http_request in http_request_pool:
        if http_request.get_http_client_status() == HTTPClient.STATUS_DISCONNECTED:
            return http_request
    return null

func _make_request(method: String, endpoint: String, body: Dictionary = {}) -> Dictionary:
    # 获取可用的 HTTPRequest
    var http_request = _get_available_http_request()

    # 如果没有可用的，等待一会儿再试
    var retry_count = 0
    while http_request == null and retry_count < 10:
        await get_tree().create_timer(0.1).timeout
        http_request = _get_available_http_request()
        retry_count += 1

    if http_request == null:
        return {
            "success": false,
            "error": "所有 HTTP 请求节点都在忙"
        }

    # 发起请求
    var error = http_request.request(url, headers, _get_http_method(method), request_body)
    # ...
```

**优点**:
- 支持多个并发请求（最多 5 个）
- 自动管理节点状态
- 如果所有节点都在忙，会等待并重试
- 不需要修改 API 调用代码

**缺点**:
- 增加了内存开销（5 个 HTTPRequest 节点）
- 如果并发请求超过 5 个，仍然可能失败

### 方案 2: Meta 值安全访问（已实施）

使用 `has_meta` 检查 meta 值是否存在，然后再使用 `get_meta` 获取。

**修复前**:
```gdscript
var bound_agent_id = current_character.get_meta("bound_agent_id", null)
```

**修复后**:
```gdscript
var bound_agent_id = null
if current_character.has_meta("bound_agent_id"):
    bound_agent_id = current_character.get_meta("bound_agent_id")
```

**优点**:
- 符合 Godot 4 的 API 规范
- 不会抛出错误
- 代码更清晰

## 技术细节

### HTTPRequest 状态检查

使用 `get_http_client_status()` 方法检查 HTTPRequest 的状态：

```gdscript
if http_request.get_http_client_status() == HTTPClient.STATUS_DISCONNECTED:
    # 节点空闲，可以使用
```

**HTTPClient 状态枚举**:
- `STATUS_DISCONNECTED` (0): 未连接，空闲状态
- `STATUS_RESOLVING` (1): 正在解析域名
- `STATUS_CONNECTING` (2): 正在连接
- `STATUS_CONNECTED` (3): 已连接
- `STATUS_REQUESTING` (4): 正在发送请求
- `STATUS_BODY` (5): 正在接收响应体

### 等待重试机制

如果所有 HTTPRequest 节点都在忙，等待 0.1 秒后重试，最多重试 10 次（总共 1 秒）：

```gdscript
var retry_count = 0
while http_request == null and retry_count < 10:
    await get_tree().create_timer(0.1).timeout
    http_request = _get_available_http_request()
    retry_count += 1
```

### Meta 值安全访问模式

Godot 4 推荐的 meta 值访问模式：

```gdscript
# 检查是否存在
if node.has_meta("key"):
    # 获取值
    var value = node.get_meta("key")
else:
    # 使用默认值
    var value = default_value
```

## 性能影响

### HTTP 请求池

**内存开销**:
- 每个 HTTPRequest 节点约占用 1-2KB 内存
- 5 个节点总共约 5-10KB
- 对于现代设备来说，影响可以忽略不计

**性能提升**:
- 支持最多 5 个并发请求
- 减少了请求等待时间
- 提升了用户体验

### Meta 值检查

**性能影响**:
- `has_meta` 是一个 O(1) 操作
- 性能影响可以忽略不计

## 后续优化建议

### 1. 动态调整请求池大小

根据实际需求动态调整请求池大小：

```gdscript
func set_max_concurrent_requests(count: int):
    if count > http_request_pool.size():
        # 增加节点
        for i in range(count - http_request_pool.size()):
            var http_request = HTTPRequest.new()
            add_child(http_request)
            http_request_pool.append(http_request)
    elif count < http_request_pool.size():
        # 减少节点
        for i in range(http_request_pool.size() - count):
            var http_request = http_request_pool.pop_back()
            http_request.queue_free()
```

### 3. 请求缓存

对于不经常变化的数据（如 Agent 列表），可以添加缓存：

```gdscript
var cache: Dictionary = {}
var cache_ttl: Dictionary = {}

func _make_request_with_cache(method: String, endpoint: String, body: Dictionary = {}, ttl: float = 60.0) -> Dictionary:
    var cache_key = "%s:%s" % [method, endpoint]

    # 检查缓存
    if cache.has(cache_key) and Time.get_ticks_msec() < cache_ttl[cache_key]:
        return cache[cache_key]

    # 发起请求
    var result = await _make_request(method, endpoint, body)

    # 缓存结果
    if result.success:
        cache[cache_key] = result
        cache_ttl[cache_key] = Time.get_ticks_msec() + ttl * 1000

    return result
```

### 4. 请求取消

支持取消正在进行的请求：

```gdscript
func cancel_request(http_request: HTTPRequest):
    if http_request.get_http_client_status() != HTTPClient.STATUS_DISCONNECTED:
        http_request.cancel_request()
```
