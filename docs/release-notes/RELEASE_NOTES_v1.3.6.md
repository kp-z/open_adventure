# Release Notes v1.3.6

**发布日期**: 2026-03-11
**版本**: v1.3.6

## 本次更新重点

本版本主要修复了后端数据库连接泄漏导致的服务卡死问题，并增强了系统监控能力。

## 🔴 严重问题修复

- **修复后端数据库连接泄漏问题**: 修复了 Agent Session 清理任务中错误的数据库连接管理方式，防止连接池耗尽导致后端无响应
- **修复定时任务连接管理**: 将 `async for db in get_db()` 改为正确的 `async with AsyncSessionLocal() as db` 模式

## 🟡 中等问题修复

- **改进错误处理**: 在定时任务中添加详细的异常追踪和错误恢复机制
- **优化连接池配置**: 针对SQLite的NullPool和其他数据库的连接池进行了优化配置

## ✨ 新增功能

- **连接池状态监控**: 新增 `get_connection_pool_status()` 函数，实时监控数据库连接池使用情况
- **增强健康检查**: `/api/system/health/detailed` 接口现在包含连接池状态、WebSocket连接数和Terminal会话统计
- **自动警告机制**: 当连接池使用率超过75%时自动记录警告日志

## 📝 文档更新

- 更新了数据库连接管理的最佳实践文档
- 添加了系统监控和故障排查指南

## 技术改进

- **连接管理优化**: 使用context manager确保数据库连接正确释放
- **监控增强**: 定时任务每5分钟记录连接池状态，便于问题诊断
- **错误追踪**: 添加详细的异常堆栈信息，提高问题定位效率

## 兼容性说明

- 本版本向后兼容，无破坏性变更
- 数据库schema无变化，可直接升级
- API接口保持兼容

## 升级指南

1. 停止当前服务
2. 替换二进制文件或更新源码
3. 重新启动服务
4. 验证 `/api/system/health/detailed` 接口正常工作
