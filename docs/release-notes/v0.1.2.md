# Claude Manager v0.1.2 发布说明

**发布日期**: 2026-02-27
**版本号**: v0.1.2
**状态**: 已发布

## 概述
Claude Manager v0.1.2 是一个功能增强版本，主要聚焦于实时执行监控、Agent 测试能力和文档结构优化。

## 新功能

### 1. 实时执行监控系统
- **WebSocket 实时更新**: 实现了基于 WebSocket 的执行状态实时推送，延迟 < 500ms
- **统一执行历史**: Agent 测试和 Workflow 执行统一展示在 Dashboard
- **后台执行支持**: 用户可以离开页面，执行继续进行
- **可视化监控**: Dashboard 高亮显示运行中的任务，右下角实时监控面板

### 2. Agent 测试能力
- **Agent 测试 API**: 新增 `/api/agents/{agent_id}/test` 端点
- **流式测试输出**: 支持 Server-Sent Events (SSE) 实时返回测试结果
- **测试执行记录**: 所有 Agent 测试执行自动记录到 Execution 表
- **测试状态追踪**: 支持 pending/running/completed/failed 状态管理

### 3. 团队管理功能
- **AgentTeam CRUD**: 完整的团队创建、编辑、删除功能
- **团队成员管理**: 支持添加/移除团队成员
- **团队 API 集成**: 前后端完整的团队管理 API 对接

### 4. 文档结构优化
- **文档重构**: 重新组织 docs/ 目录结构
- **设计资源迁移**: 将 Figma 设计资源迁移到 `docs/design/figma-assets/`
- **发布说明规范**: 建立标准化的发布说明文档格式

## 改进

### 性能优化
- 优化 WebSocket 连接管理，减少资源占用
- 改进数据库查询性能，加快执行历史加载速度

### 用户体验
- Dashboard 界面优化，更清晰的执行状态展示
- 改进错误提示信息，更友好的用户反馈

### 代码质量
- 移除旧的 Agent 定义文件，清理冗余代码
- 更新依赖包版本，修复安全漏洞
- 统一代码风格，改进代码可维护性

## 技术细节

### 数据库变更
- 扩展 `Execution` 模型，新增 `execution_type` 字段支持 Agent 测试
- 添加 `agent_id` 外键关联 Agent 测试执行

### API 变更
- 新增 WebSocket 端点: `/ws/executions/{execution_id}`
- 新增 Agent 测试端点: `/api/agents/{agent_id}/test`
- 更新 CORS 配置，支持跨域 WebSocket 连接

### 前端变更
- 集成 WebSocket 客户端，实现实时状态更新
- 新增 AgentTeam 管理页面和组件
- 优化 Dashboard 执行历史展示

## 已知问题
- WebSocket 连接在某些网络环境下可能不稳定
- Agent 测试输出较大时可能影响性能

## 升级指南

### 从 v0.1.1 升级
1. 拉取最新代码: `git pull origin main`
2. 更新后端依赖: `cd backend && pip install -e .`
3. 运行数据库迁移: `alembic upgrade head`
4. 更新前端依赖: `cd frontend && npm install`
5. 重启服务

### 数据库迁移
```bash
cd backend
alembic upgrade head
```
