# Release Notes - v0.1.5

**发布日期**: 2026-02-28

## 🎉 新功能

### Terminal 会话持久化
- **会话保持**: 页面刷新或切换后，Terminal 会话继续运行，不会丢失工作状态
- **智能清理**: 移除基于时间的自动清理机制，只清理已死亡的进程
- **会话恢复**: 支持通过 session ID 重新连接到已存在的会话（基础设施已就绪）

### Terminal UI 改进
- **项目名命名**: Terminal tab 以项目名显示，更易识别（如 "open_adventure", "steady"）
- **移动端优化**:
  - Tab 显示在 terminal 上方，左对齐
  - 隐藏分屏按钮，优化移动端体验
- **界面简化**: 移除全屏按钮，保持界面简洁

### API 增强
- **Session 管理 API**:
  - `GET /api/terminal/status` - 查看所有活跃的 terminal 会话
  - `POST /api/terminal/sessions/{id}/close` - 手动关闭指定会话
- **会话信息**: 返回进程 PID、运行状态、创建时间等详细信息

## 🐛 Bug 修复

- 修复自动进入 claude code 界面的逻辑错误
- 修复 Terminal 创建时的变量作用域问题（UnboundLocalError）
- 修复 Terminal tab 切换时的渲染问题

## 🔧 技术改进

- 使用 UUID 作为 session ID，替代 WebSocket 对象 ID
- 添加进程存活检测（`is_process_alive()`）
- WebSocket 断开时保持会话运行，支持重连
- 优化 Terminal 组件的 React key 管理

## 📝 升级说明

## ⚠️ 注意事项

- Terminal 会话持久化功能需要后端持续运行
- 关闭浏览器后，后端的 terminal 进程会继续运行
- 建议定期检查 `/api/terminal/status` 清理不需要的会话
