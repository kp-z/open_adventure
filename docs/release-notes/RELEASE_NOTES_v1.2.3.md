# Release Notes v1.2.3

**发布日期**: 2026-03-03

## 本次更新重点

本次更新实现了**源码保护的二进制打包方案**，使用 PyInstaller 将 Python 源码编译为字节码，防止源码泄露。同时更新了发布流程规范，支持二进制和源码两种发布方式。

## ✨ 新增功能

### 二进制打包支持
- **源码保护**: 使用 PyInstaller 将所有 Python 代码编译为字节码并打包到 PYZ 文件中
- **独立运行**: 二进制包包含 Python 解释器和所有依赖，用户无需安装 Python 环境
- **自动化构建**: 新增 `scripts/build_binary.sh` 脚本，支持 macOS 和 Linux 平台自动化构建
- **跨平台支持**: 构建脚本自动检测平台和架构（macOS ARM64、Linux x86_64）

### 构建脚本功能
- 自动检查前端构建产物
- 自动激活 Python 虚拟环境
- 自动清理旧的构建产物
- 验证源码保护效果（确保无 .py 文件）
- 自动打包为 tar.gz 压缩包
- 详细的构建信息输出

## 🔧 技术改进

### PyInstaller 配置修复
- **修复源码泄露问题**: 删除了错误的源码收集代码，确保源码被编译而非直接复制
- **修复路径问题**: 使用 `SPECPATH` 替代 `__file__` 变量，解决 PyInstaller 执行时的路径问题
- **优化数据收集**: 只收集前端静态文件和数据库模板，不收集源码文件

### 发布流程优化
- 更新 CLAUDE.md 中的 Release 发布规范

- 压缩包命名统一为 `open_adventure-v{版本号}-{平台}-{架构}.tar.gz`

## 🚀 使用方法

### 二进制版本（推荐）

1. 下载并解压：
```bash
tar -xzf open_adventure-v1.2.3-macos-arm64.tar.gz
cd open-adventure
```

2. 运行：
```bash
./open-adventure
```

### 构建二进制版本

在项目根目录运行：
```bash
./scripts/build_binary.sh
```

## 🔄 升级指南

从 v1.2.2 升级到 v1.2.3：
