## 修复 2: HTTP 请求并发冲突

### 问题描述
多个 HTTP 请求同时发起时，出现以下错误：

1. **HTTPRequest 并发冲突**
   ```
   ERROR: HTTPRequest is processing a request. Wait for completion or cancel it before attempting a new one.
   ```
   - 错误码: 44 (ERR_BUSY)
   - 原因: 只有一个 HTTPRequest 节点

2. **Meta 值访问错误**
   ```
   ERROR: The object does not have any 'meta' values with the key 'bound_agent_id'.
   ```
   - 原因: 使用了 Godot 4 不支持的 get_meta 两参数形式

3. **AgentDialogSystem 缺失**
   ```
   [CharacterManager] 错误: 找不到 AgentDialogSystem
   ```
   - 原因: 场景中缺少 AgentDialogSystem 节点

### 解决方案

#### 1. HTTP 请求池
实现了一个 HTTP 请求节点池，支持最多 5 个并发请求：

```gdscript
# HTTP 请求节点池
var http_request_pool: Array = []
var max_concurrent_requests = 5

func _ready():
    # 创建 HTTPRequest 节点池
    for i in range(max_concurrent_requests):
        var http_request = HTTPRequest.new()
        add_child(http_request)
        http_request_pool.append(http_request)

func _get_available_http_request() -> HTTPRequest:
    """获取可用的 HTTPRequest 节点"""
    for http_request in http_request_pool:
        if http_request.get_http_client_status() == HTTPClient.STATUS_DISCONNECTED:
            return http_request
    return null
```

**特性**:
- 支持最多 5 个并发请求
- 自动检测节点状态
- 智能等待重试机制（0.1 秒间隔，最多 10 次）
- 零配置，自动管理

#### 2. Meta 值安全访问
修复了不安全的 get_meta 调用：

**修复前**:
```gdscript
var bound_agent_id = current_character.get_meta("bound_agent_id", null)
```

**修复后**:
```gdscript
var bound_agent_id = null
if current_character.has_meta("bound_agent_id"):
    bound_agent_id = current_character.get_meta("bound_agent_id")
```
