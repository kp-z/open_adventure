## 问题描述

用户报告 Linux 兼容版（Ubuntu 20.04 基础）在启动时出现以下错误：

```
TypeError: unsupported operand type(s) for |: 'type' and 'NoneType'
```

错误发生在：
- `backend/app/database/migration.py:33`
- `backend/app/core/tag_definitions.py:104`

### 影响范围
- **仅影响 Linux 兼容版**（Ubuntu 20.04，Python 3.8）
- macOS 版本（Python 3.11）和 Linux 最新版（Python 3.12）不受影响
- 导致服务无法启动，完全阻塞用户使用

## 问题根源

### 1. Python 版本差异

Linux 兼容版使用 Ubuntu 20.04 的默认 Python 3.8，而 Python 3.8 不支持 PEP 604 的新式类型注解语法。

### 2. 不兼容的语法

以下语法在 Python 3.10+ 中可用，但在 Python 3.8 中会报错：

```python
# ❌ Python 3.8 不支持
def get_current_revision(database_url: str) -> str | None:
    pass

def validate_tags(tags: List[str]) -> tuple[bool, str]:
    pass

def get_tag_dimension(tag: str) -> TagDimension | None:
    pass
```

### 3. `from __future__ import annotations` 的局限性

虽然这两个文件都使用了 `from __future__ import annotations`，但这只延迟注解的求值，不改变运行时行为。当类型注解在运行时被求值时（如 Pydantic 模型验证），仍会报错。

## 修复方案

### 1. 修复 `backend/app/database/migration.py`

**修改前**：
```python
def get_current_revision(database_url: str) -> str | None:
    """获取当前数据库的迁移版本"""
    # ...
```

**修改后**：
```python
from typing import Optional

def get_current_revision(database_url: str) -> Optional[str]:
    """获取当前数据库的迁移版本"""
    # ...
```

### 2. 修复 `backend/app/core/tag_definitions.py`

**修改前**：
```python
from typing import Dict, List, Set

def validate_tags(tags: List[str]) -> tuple[bool, str]:
    # ...

def get_tag_dimension(tag: str) -> TagDimension | None:
    # ...
```

**修改后**：
```python
from typing import Dict, List, Set, Optional, Tuple

def validate_tags(tags: List[str]) -> Tuple[bool, str]:
    # ...

def get_tag_dimension(tag: str) -> Optional[TagDimension]:
    # ...
```

### 3. 全局检查

使用 grep 搜索所有可能的不兼容语法：

```bash
grep -r "-> .*|.*:" backend/app --include="*.py"
grep -r "-> tuple\[" backend/app --include="*.py"
grep -r "-> list\[" backend/app --include="*.py"
grep -r "-> dict\[" backend/app --include="*.py"
```

结果：未发现其他文件使用不兼容的语法。

## 经验总结

### 1. Python 版本兼容性

**问题**：
- 新式类型注解（PEP 604）在 Python 3.10+ 中引入
- 使用 `from __future__ import annotations` 不能完全解决运行时兼容性问题

**解决方案**：
- 对于需要支持 Python 3.8+ 的项目，始终使用 `typing` 模块的类型
- 使用 `Optional[T]` 而不是 `T | None`
- 使用 `Tuple[T1, T2]` 而不是 `tuple[T1, T2]`
- 使用 `List[T]` 而不是 `list[T]`
- 使用 `Dict[K, V]` 而不是 `dict[K, V]`

### 2. 多平台构建测试

**问题**：
- 在 macOS（Python 3.11）上开发，未发现 Python 3.8 兼容性问题
- Linux 兼容版使用 Docker 构建，但构建成功不代表运行时兼容

**改进建议**：
- 在 CI/CD 中添加 Python 3.8 兼容性测试
- 使用 `mypy` 或 `pyright` 进行静态类型检查
- 在不同 Python 版本的环境中运行测试

### 3. 类型注解最佳实践

**推荐做法**：
```python
from typing import Optional, Tuple, List, Dict

# ✅ 兼容 Python 3.8+
def func1() -> Optional[str]:
    pass

def func2() -> Tuple[bool, str]:
    pass

def func3() -> List[int]:
    pass

def func4() -> Dict[str, int]:
    pass
```

**避免做法**（除非确定只支持 Python 3.10+）：
```python
# ❌ 仅 Python 3.10+ 支持
def func1() -> str | None:
    pass

def func2() -> tuple[bool, str]:
    pass

def func3() -> list[int]:
    pass

def func4() -> dict[str, int]:
    pass
```
