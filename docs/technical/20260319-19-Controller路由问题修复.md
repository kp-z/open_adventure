# Controller 路由问题修复

## 问题描述
访问 `http://localhost:5173/projects/controller/` 时出现 "Route Error"。

## 原因分析
1. React Router 中有 `/controller` 路由，指向旧的 React 组件
2. Vite PWA 的 `navigateFallback` 将所有路由重定向到 `/index.html`
3. Vite 开发服务器不会自动将目录路径重定向到 `index.html`
4. 导致无法访问 `public/projects/controller/` 下的静态 HTML 文件

## 修复方案

### 1. 删除 React Router 中的 controller 路由
**文件**: `frontend/src/app/routes.tsx`

删除：
```typescript
import Controller from './pages/Controller';

// ...

{
  path: 'controller',
  Component: Controller,
},
```

### 2. 更新 Vite PWA 配置
**文件**: `frontend/vite.config.ts`

添加 `/projects` 到 `navigateFallbackDenylist`：
```typescript
navigateFallbackDenylist: [
  /^\/microverse/, // 排除 microverse 路由
  /^\/projects/,   // 排除 projects 路由，让它直接访问静态 HTML
],
```

### 3. 更新 Projects 列表页面链接
**文件**: `frontend/public/projects/index.html`

修改链接为完整的 HTML 文件路径：
```html
<!-- 修改前 -->
<a href="/projects/controller/" class="project-card" target="_blank">

<!-- 修改后 -->
<a href="/projects/controller/index.html" class="project-card" target="_blank">
```

### 4. 重启前端服务
```bash
# 停止当前服务（Ctrl+C）
# 重新启动
cd frontend && npm run dev
```
