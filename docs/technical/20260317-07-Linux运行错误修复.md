## 问题背景

用户在 Linux 服务器上运行 Open Adventure v0.1.0 时遇到两个主要错误：

1. **数据库 Schema 不匹配**：`sqlite3.OperationalError: no such column: executions.process_pid`
2. **Godot 游戏界面 404**：`GET /microverse/index.html HTTP/1.1 404 Not Found`

## 错误分析

### 错误 1：数据库 Schema 不匹配（严重）

**影响范围**：
- `GET /api/executions/` - 执行历史列表（500 错误）
- `GET /api/dashboard/stats` - Dashboard 统计（500 错误）
- Agent Monitor Service - 后台监控服务失败

**根本原因**：
- Linux 环境的数据库文件（`/root/.open_adventure/open_adventure.db`）版本过旧
- 缺少以下字段：
  - `process_pid` (INTEGER)
  - `work_dir` (VARCHAR)
  - `log_file` (VARCHAR)
  - `terminal_pid` (INTEGER)
  - `terminal_command` (TEXT)
  - `terminal_cwd` (VARCHAR)
  - `terminal_output` (TEXT)

**迁移文件问题**：
- 迁移文件 `a2be1393b6ef_add_agent_framework_and_plan_models.py` 包含 SQLite 不兼容的操作：
  - `ALTER COLUMN TYPE` - SQLite 不支持（已修复）
  - `CREATE FOREIGN KEY` - SQLite 需要使用 batch mode（已修复）
- 当前迁移会停在 `eb7fe8d3da6d`，无法升级到最新版本 `99631bce18c8`

### 错误 2：Godot 游戏界面加载失败（中等）

**影响范围**：
- Godot 游戏化界面无法加载
- 但 `GET /microverse/version.json` 返回 200 OK

**根本原因**：
- Linux 服务器上用户数据目录 `~/.open_adventure/frontend_dist/microverse/` 缺少文件
- 可能是首次安装时复制不完整

## 修复方案

### 修复 1：数据库迁移文件（已完成）

**修改文件**：`backend/alembic/versions/a2be1393b6ef_add_agent_framework_and_plan_models.py`

**修改内容**：

1. **使用 batch mode 添加外键约束**（第 151-160 行）：
```python
# SQLite 需要使用 batch mode 来添加外键约束
with op.batch_alter_table('tasks', schema=None) as batch_op:
    batch_op.create_foreign_key(op.f('fk_tasks_agent_id_agents'), 'agents', ['agent_id'], ['id'])
    batch_op.create_foreign_key(op.f('fk_tasks_plan_template_id_plan_templates'), 'plan_templates', ['plan_template_id'], ['id'])
    batch_op.create_foreign_key(op.f('fk_tasks_current_plan_id_plans'), 'plans', ['current_plan_id'], ['id'])
```

2. **使用 batch mode 删除外键约束**（第 162-167 行）：
```python
# SQLite 需要使用 batch mode 来删除外键约束
with op.batch_alter_table('tasks', schema=None) as batch_op:
    batch_op.drop_constraint(op.f('fk_tasks_current_plan_id_plans'), type_='foreignkey')
    batch_op.drop_constraint(op.f('fk_tasks_plan_template_id_plan_templates'), type_='foreignkey')
    batch_op.drop_constraint(op.f('fk_tasks_agent_id_agents'), type_='foreignkey')
```

3. **注释掉 ALTER COLUMN TYPE 操作**（第 125-130 行）：
```python
# SQLite 不支持 ALTER COLUMN TYPE，跳过此操作
# execution_type 字段已经存在，类型差异不影响功能
# op.alter_column('executions', 'execution_type', ...)
```

## 技术细节

### SQLite 限制

SQLite 不支持以下操作：
1. `ALTER COLUMN TYPE` - 无法直接修改列类型
2. `CREATE FOREIGN KEY` - 需要使用 batch mode

**解决方案**：
- 使用 Alembic 的 `batch_alter_table` 上下文管理器
- 在 batch mode 中，Alembic 会：
  1. 创建新表（包含外键约束）
  2. 复制数据
  3. 删除旧表
  4. 重命名新表
