## 概述

本文档记录了 Open Adventure 项目测试模块的实现过程。测试模块提供统一的测试管理、执行和报告功能，支持 pytest、Playwright 和 Vitest 三种测试框架。

## 实现内容

### 1. 后端核心模块

#### 数据模型
- `TestNode`: 测试树节点模型
- `TestExecution`: 测试执行记录模型

#### 测试树管理器
- 从 YAML 配置加载测试树
- 获取完整的树形结构
- 递归构建树形节点

#### 测试执行引擎
- 支持三种测试框架：pytest、Playwright、Vitest
- 使用安全的命令执行方式
- 实时读取测试输出
- 解析测试结果

#### API 路由
- GET /api/testing/tree
- POST /api/testing/tree/reload
- GET /api/testing/nodes/{node_id}
- POST /api/testing/run/{node_id}
- GET /api/testing/executions
- GET /api/testing/executions/{execution_id}
- GET /api/testing/coverage

### 2. 数据库迁移

创建了数据库迁移脚本，包含：
- test_nodes 表
- test_executions 表
- 必要的索引和外键

### 3. 前端界面

#### 测试管理页面
- 左侧：测试树展示
- 右侧：测试详情和执行结果

#### 路由配置
- 添加 /testing 路由
- 添加测试菜单项到导航栏

### 4. 测试配置

定义了三个主要测试分类：
1. 后端功能测试
2. 前端功能测试
3. 端到端测试

## 使用方法

### 初始化测试树

```bash
cd backend
PYTHONPATH=/Users/kp/项目/Proj/claude_manager/backend python scripts/init_test_tree.py
```

### 访问测试页面

启动服务后访问 http://localhost:5173/testing

### 运行测试

1. 点击左侧测试树中的测试节点
2. 点击右侧的运行按钮
3. 查看测试执行结果
