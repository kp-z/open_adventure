## 已知问题

### 1. 会话恢复功能未实现 ⚠️

**问题描述**：
- 关闭浏览器后重新打开，无法自动恢复到之前的 tmux 会话
- tmux 会话信息已保存到 localStorage，但恢复逻辑未实现

**影响**：
- 用户需要手动通过 `tmux attach` 连接到会话
- 降低了持久化功能的用户体验

**优先级**: 高

**建议解决方案**：
1. 在 Terminal 页面加载时，从 localStorage 读取 tmux 会话信息
2. 调用 `/api/terminal/tmux/sessions` 检查会话是否存活
3. 如果存活，提示用户是否恢复
4. 实现重连到 tmux 会话的逻辑

### 2. 会话超时清理机制缺失 ⚠️

**问题描述**：
- 长时间未使用的 tmux 会话会一直占用资源
- 没有自动清理机制

**影响**：
- 可能导致资源浪费
- 僵尸会话累积

**优先级**: 中

**建议解决方案**：
1. 添加会话最后活动时间记录
2. 定期检查并清理超时会话（如 24 小时未活动）
3. 清理前通知用户
