## 核心规则亮点

### 1. 不可变性原则 (CRITICAL)
```typescript
// WRONG: 修改原对象
function updateUser(user: User, name: string): User {
  user.name = name // MUTATION!
  return user
}

// CORRECT: 返回新对象
function updateUser(user: Readonly<User>, name: string): User {
  return {
    ...user,
    name
  }
}
```

### 2. 文件组织原则
- **多个小文件 > 少数大文件**
- 典型文件大小：200-400 行
- 最大文件大小：800 行
- 按功能/领域组织，而非按类型

### 3. 错误处理原则
- 在每个层级显式处理错误
- UI 层提供用户友好的错误消息
- 服务端记录详细的错误上下文
- 永远不要静默吞掉错误

### 4. 输入验证原则
- 在系统边界验证所有输入
- 使用基于 Schema 的验证（如 Zod）
- 快速失败，提供清晰的错误消息
- 永远不要信任外部数据

### 5. TypeScript 类型原则
- 为公共 API 添加显式类型
- 避免使用 `any`，使用 `unknown` 代替
- 使用 `interface` 定义可扩展的对象形状
- 使用 `type` 定义联合类型、交叉类型等
