# 移动端滚动条测试指南

## 快速测试步骤

### 1. 访问应用
- **本地访问**: http://localhost:5173
- **网络访问**: http://10.12.127.151:5173 (可用手机访问)

### 2. 进入 Terminal 页面
1. 打开应用后，点击左侧导航栏的 "TERMINAL" 
2. 创建一个新的终端会话
3. 在终端中输入一些命令产生足够的输出内容

### 3. 测试移动端滚动条

#### 在手机上测试（推荐）
1. 用手机浏览器访问: http://10.12.127.151:5173
2. 进入 Terminal 页面
3. 创建终端并执行以下命令产生长输出：
   ```bash
   # 产生大量输出
   ls -la /
   cat /etc/passwd
   find /usr -name "*.txt" | head -50
   ```

#### 在桌面浏览器模拟移动端
1. 打开 Chrome 开发者工具 (F12)
2. 点击设备模拟按钮 (📱 图标)
3. 选择移动设备 (如 iPhone 12 Pro)
4. 刷新页面

### 4. 验证功能

#### 滚动条显示
- ✅ 当终端内容超过屏幕高度时，右侧应出现滚动条
- ✅ 滚动条应该是半透明的黑色背景，白色拇指
- ✅ 桌面端不应显示滚动条

#### 触摸交互
- ✅ 触摸拖拽拇指可以滚动终端内容
- ✅ 点击滚动条任意位置可以快速跳转
- ✅ 拖拽时显示滚动百分比提示

#### 自动隐藏
- ✅ 3秒无操作后滚动条自动隐藏
- ✅ 有滚动活动时滚动条重新显示

#### 响应式行为
- ✅ 终端内容变化时滚动条实时更新
- ✅ 屏幕旋转时滚动条正确调整
- ✅ 键盘弹出时滚动条位置正确

## 测试命令推荐

### 产生长输出的命令
```bash
# 1. 显示目录内容
ls -la /usr/bin

# 2. 显示系统信息
ps aux

# 3. 显示文件内容
cat /etc/hosts

# 4. 循环输出
for i in {1..50}; do echo "Line $i: This is a test line to create scrollable content"; done

# 5. 显示环境变量
env

# 6. 显示网络连接
netstat -an 2>/dev/null || ss -an
```

## 常见问题排查

### 滚动条不显示
1. 确认是在移动端或移动端模拟模式
2. 确认终端内容足够长（超过屏幕高度）
3. 检查浏览器控制台是否有错误

### 触摸不响应
1. 确认是在真实的触摸设备上测试
2. 尝试刷新页面
3. 检查是否有其他元素遮挡滚动条

### 滚动不流畅
1. 确认设备性能足够
2. 尝试减少终端输出内容
3. 检查网络连接是否稳定
