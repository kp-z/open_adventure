# Terminal 自动重连功能实现

**创建日期**: 2026-03-10
**状态**: 已完成

## 概述

实现了 Terminal 页面的自动重连功能，解决了移动端长时间不使用后（例如手机息屏或切换到其他应用）WebSocket 断开的问题。

## 实现内容

### 1. 后端 API 增强

#### 新增 Claude 状态检测 API

**文件**: `backend/app/api/terminal.py`

```python
@router.get("/session/{session_id}/claude-status")
async def get_claude_status(session_id: str):
    """检查指定 session 中的 Claude 进程是否还在运行"""
    if session_id not in sessions:
        return JSONResponse({
            "running": False,
            "session_exists": False,
            "process_alive": False,
            "claude_resume_session": None,
        })

    session = sessions[session_id]
    claude_running = session.check_claude_running()

    return JSONResponse({
        "running": claude_running,
        "session_exists": True,
        "process_alive": session.is_process_alive(),
        "claude_resume_session": session.claude_resume_session if hasattr(session, 'claude_resume_session') else None,
    })
```

**功能**:
- 检查 terminal session 是否存在
- 检查 PTY 进程是否还在运行
- 检查 Claude 进程是否还在运行
- 返回 Claude 会话恢复的 session ID（如果有）

### 2. 前端重连机制

#### TerminalContext 增强

**文件**: `frontend/src/app/contexts/TerminalContext.tsx`

**新增字段**:
```typescript
export interface TerminalInstance {
  // ... 原有字段
  reconnecting: boolean;           // 是否正在重连
  reconnectAttempts: number;       // 重连尝试次数
  maxReconnectAttempts: number;    // 最大重连次数（默认 3）
}
```

**新增函数**:
- `reconnectTerminal(terminalId: string): Promise<boolean>` - 重连指定终端
- `checkClaudeStatus(sessionId: string)` - 检查 Claude 状态

**重连逻辑**:
1. 检查终端是否存在且有 session ID
2. 检查是否已在重连中
3. 关闭旧的 WebSocket 连接
4. 创建新的 WebSocket 连接到同一个 session_id
5. 等待连接建立（5秒超时）
6. 重新设置 WebSocket 事件处理
7. 发送 `restore_ready` 消息，请求恢复 scrollback
8. 检查 Claude 进程状态
9. 如果连接失败，使用指数退避策略重试（1s, 2s, 4s）

**指数退避策略**:
- 第 1 次重试：延迟 1 秒
- 第 2 次重试：延迟 2 秒
- 第 3 次重试：延迟 4 秒
- 超过 3 次后放弃，提示用户重新创建终端

### 3. 页面可见性检测

**文件**: `frontend/src/app/pages/Terminal.tsx`

```typescript
// 页面可见性检测 - 自动重连
useEffect(() => {
  if (!isMobile) return;

  const handleVisibilityChange = () => {
    if (!document.hidden) {
      // 页面变为可见，检查所有终端连接
      console.log('[Terminal] Page became visible, checking terminal connections...');
      terminals.forEach((terminal) => {
        if (terminal.ws.readyState !== WebSocket.OPEN && !terminal.reconnecting) {
          console.log(`[Terminal] Terminal ${terminal.id} is disconnected, attempting reconnect...`);
          reconnectTerminal(terminal.id);
        }
      });
    }
  };

  document.addEventListener('visibilitychange', handleVisibilityChange);
  return () => document.removeEventListener('visibilitychange', handleVisibilityChange);
}, [terminals, isMobile, reconnectTerminal]);
```

**功能**:
- 监听 `visibilitychange` 事件
- 当页面从隐藏变为可见时，检查所有终端的连接状态
- 如果 WebSocket 已断开且未在重连中，触发重连

### 4. WebSocket 自动重连

**在 WebSocket `onclose` 事件中**:
```typescript
newWs.onclose = (event) => {
  logLifecycle(terminal, 'disposed', `ws close:${event.code}`);
  if (event.code !== 1000) {
    terminal.term.writeln('\x1b[31m✗ Connection closed\x1b[0m');
    // 自动重连
    if (!terminal.reconnecting && terminal.reconnectAttempts < terminal.maxReconnectAttempts) {
      console.log('[TerminalContext] WebSocket closed, attempting reconnect...');
      setTimeout(() => {
        reconnectTerminal(terminal.id);
      }, 1000);
    }
  }
};
```

**功能**:
- 当 WebSocket 非正常关闭时（code !== 1000）
- 如果未在重连中且未超过最大重连次数
- 延迟 1 秒后自动触发重连

## 用户体验优化

### 1. 重连状态提示

- **重连中**: 终端显示 "正在重连..." 状态
- **重连成功**: 显示成功通知 "终端已重新连接"
- **重连失败**: 显示错误通知 "终端重连失败，已达到最大重试次数"

### 2. Claude 会话恢复提示

当重连成功后，如果检测到 Claude 进程已退出但有会话可以恢复：
- 显示通知："检测到 Claude 会话已断开，是否恢复？"
- 用户可以选择恢复会话（执行 `claude --resume`）

### 3. Scrollback 恢复

重连成功后：
- 自动发送 `restore_ready` 消息
- 后端回放最近的终端输出（最多 512KB）
- 用户可以看到断开前的终端历史

## 技术细节

### WebSocket 重连策略

- **指数退避**: 1s, 2s, 4s
- **最大重连次数**: 3 次
- **重连时复用原有的 session_id**

### 页面可见性检测

```typescript
document.addEventListener('visibilitychange', handleVisibilityChange);
```

- 监听 `visibilitychange` 事件
- 仅在移动端启用（`isMobile` 为 true）
- 页面变为可见时检查所有终端连接

### 后端 Session 保持

- WebSocket 断开后不关闭 PTY session
- Session 保持在内存中，直到用户主动关闭或超时
- 重连时直接复用现有 session

## 注意事项

1. **安全性**: 确保只有同一用户可以重连到自己的 session
2. **资源管理**: 设置 session 超时时间，避免内存泄漏
3. **错误处理**: 重连失败时提供清晰的错误信息和恢复建议
4. **用户体验**: 重连过程中显示加载状态，避免用户困惑
