# 游戏预加载功能验证指南

**创建日期**: 2026-03-18
**章节编号**: 12
**状态**: 已完成

## 验证步骤

### 1. 清除缓存

在浏览器控制台执行：

```javascript
localStorage.removeItem('open-adventure-initialized');
localStorage.removeItem('microverse_game_loaded');
location.reload();
```

### 2. 观察初始化过程

刷新页面后，打开浏览器控制台（F12），观察以下日志：

- `[Preload] 开始预加载 Microverse...` - 预加载开始
- `[Preload] Microverse 预加载完成` - 预加载完成（约 3-5 秒后）

### 3. 检查 localStorage

在控制台执行：

```javascript
console.log('microverse_game_loaded:', localStorage.getItem('microverse_game_loaded'));
```

预期结果：`microverse_game_loaded: true`

### 4. 进入游戏模式

点击左侧导航栏的"游戏模式"按钮，观察：

- **预加载成功**：直接显示游戏，无加载界面
- **预加载未完成**：显示加载界面（通常只需 1-2 秒）

## 预期效果

### 首次进入系统

1. 系统初始化完成后，后台自动开始预加载游戏
2. 预加载过程不阻塞用户操作
3. 预加载完成后，`localStorage` 中有 `microverse_game_loaded=true`

### 进入游戏模式

- **预加载完成**：直接显示游戏，无需等待
- **预加载未完成**：显示加载界面，等待游戏加载完成

## 边界情况测试

### 1. 预加载失败

模拟网络断开：

```javascript
// 在控制台执行
localStorage.removeItem('microverse_game_loaded');
// 断开网络连接
// 刷新页面
```

预期：系统初始化正常完成，预加载失败不影响其他功能

### 2. 游戏版本更新

修改 `/microverse/version.json` 的 `exportTime`：

```bash
cd /Users/kp/项目/Proj/claude_manager/microverse
# 修改 version.json
```

预期：刷新页面后，缓存被清除，游戏重新加载

### 3. 用户不使用游戏模式

首次进入系统后，不进入游戏模式，观察：

- 预加载在后台进行
- 不影响其他页面的使用
- 控制台有预加载日志

## 性能指标

- **预加载时间**：3-5 秒（取决于网络速度）
- **首次加载时间**：无明显增加（后台预加载）
- **进入游戏时间**：< 1 秒（预加载完成后）

## 回滚方案

如果预加载功能出现问题，可以快速回滚：

1. 删除 `InitializationContext.tsx` 中的 `preloadMicroverse()` 调用
2. 删除 `preloadMicroverse` 函数定义
3. 删除 `MICROVERSE_LOADED_KEY` 常量
4. 重新构建前端：`cd frontend && npm run build`
