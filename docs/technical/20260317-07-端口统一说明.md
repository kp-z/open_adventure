# 端口统一说明

**创建日期**：2026-03-17
**状态**：已完成

## 变更概述

Open Adventure 已完成端口统一，所有服务（前端、后端、Microverse 游戏）现在统一使用 **38080** 端口。

## 变更前后对比

### 变更前（旧架构）
- 后端 API：`http://localhost:8000` 或 `http://localhost:38080`
- 前端开发服务器：`http://localhost:5173`
- Microverse 游戏：`http://localhost:5174`

**问题**：
- 多个端口，配置复杂
- 需要处理 CORS 跨域问题
- 部署时需要分别配置前后端

### 变更后（新架构）
- **统一端口**：`http://localhost:38080`
- 主界面：`http://localhost:38080`
- API 文档：`http://localhost:38080/docs`
- Microverse 游戏：`http://localhost:38080/microverse`

**优势**：
- 单一端口，配置简单
- 无 CORS 问题（同源）
- 部署简单，只需启动后端
- 符合生产环境最佳实践

## 技术实现

### 后端静态文件服务

后端 `main.py` 已配置静态文件服务：

```python
# 挂载静态文件目录
app.mount("/assets", StaticFiles(directory=os.path.join(FRONTEND_DIR, "assets")), name="assets")
app.mount("/avatars", StaticFiles(directory=os.path.join(FRONTEND_DIR, "avatars")), name="avatars")
app.mount("/images", StaticFiles(directory=os.path.join(FRONTEND_DIR, "images")), name="images")
app.mount("/microverse", StaticFiles(directory=os.path.join(FRONTEND_DIR, "microverse"), html=True), name="microverse")

# SPA catch-all 路由
@app.get("/{full_path:path}")
async def serve_spa(full_path: str):
    # 将所有非 API 请求重定向到 index.html
    return FileResponse(os.path.join(FRONTEND_DIR, "index.html"))
```

### 前端构建

前端使用 Vite 构建为静态文件，输出到 `frontend/dist/`：

```bash
cd frontend
npm run build
```

构建产物：
- `frontend/dist/index.html` - 主页面
- `frontend/dist/assets/` - JS/CSS 资源
- `frontend/dist/microverse/` - Godot 游戏文件

### Godot 游戏集成

Godot 游戏导出到 `frontend/public/microverse/`，构建时会复制到 `frontend/dist/microverse/`：

```bash
cd microverse
godot --headless --export-release "Web" ../frontend/public/microverse/index.html
```

## 开发注意事项

### 前端开发

如果需要使用 Vite 的热重载功能进行前端开发：

```bash
cd frontend
npm run dev
```

这会启动开发服务器在 `http://localhost:5173`，但**仅用于开发调试**。

**重要**：
- 开发完成后，必须运行 `npm run build` 构建生产版本
- 生产环境和测试环境都应该访问 `http://localhost:38080`

### API 调用

前端代码中的 API 调用使用相对路径：

```typescript
// 正确：使用相对路径
fetch('/api/agents')

// 错误：不要硬编码端口
fetch('http://localhost:38080/api/agents')
```

这样可以确保在任何端口下都能正常工作。

### Godot 游戏 API

Godot 游戏中的 API 调用已更新为 38080 端口：

```gdscript
# microverse/script/api/MicroverseAPIClient.gd
const BASE_URL = "http://localhost:38080/api"
```

## 迁移指南

如果你有旧的配置或脚本引用了 5173 端口，请按以下步骤迁移：

### 1. 更新访问地址
- 将 `http://localhost:5173` 改为 `http://localhost:38080`
- 将 `http://localhost:8000` 改为 `http://localhost:38080`

### 2. 移除前端环境变量
删除或注释 `frontend/.env.local` 中的以下配置（不再需要）：
```bash
# VITE_API_BASE_URL=...  # 不再需要
# VITE_WS_BASE_URL=...   # 不再需要
```

### 3. 更新防火墙规则
- 移除 5173 端口的防火墙规则
- 确保 38080 端口允许访问

### 4. 更新文档和脚本
检查你的自定义文档和脚本，将所有 5173 端口引用改为 38080。

## 常见问题

### Q: 为什么还能访问 http://localhost:5173？
A: 如果你运行了 `npm run dev`，Vite 开发服务器会在 5173 端口启动。这仅用于开发调试，生产环境不应使用。

### Q: 如何完全禁用 5173 端口？
A: 不要运行 `npm run dev`，只使用 `./start.sh` 启动后端服务。

### Q: 局域网访问时需要配置什么？
A: 无需特殊配置，只需确保防火墙允许 38080 端口，然后访问 `http://<局域网IP>:38080`。

### Q: 如何更改端口？
A: 使用环境变量或命令行参数：
```bash
PORT=38081 ./start.sh
# 或
./open-adventure --port 38081
```
