## 用户临时解决方案

在下个版本发布前，用户可以使用以下临时方案：

### 方案 1：删除旧数据库（会丢失数据）

```bash
./stop.sh
rm ~/.open_adventure/open_adventure.db
./start.sh
```

### 方案 2：等待下个版本（推荐）

下个版本（v0.2.0）将包含所有修复，用户只需：

```bash
./scripts/upgrade.sh
```
