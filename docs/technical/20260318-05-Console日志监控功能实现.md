## 概述

在测试页面（testing.html）添加了 Console 日志监控功能，可以自动捕获并保存来自主应用、Microverse 游戏和后端的日志，帮助开发者进行调试和问题排查。

## 需求背景

用户希望在测试页面添加一个功能：
1. 自动监控 console 输出的日志
2. 保存到本地文件系统
3. 分门别类管理
4. 帮助本地 AI 进行 debug

#### WebSocket 实时日志流

```python
@router.websocket("/stream")
async def websocket_endpoint(websocket: WebSocket):
    """实时日志流 WebSocket"""
    await websocket.accept()
    active_connections.append(websocket)

    try:
        while True:
            # 接收来自客户端的日志
            data = await websocket.receive_text()
            log_entry = json.loads(data)

            # 写入文件
            await save_log_entry(log_entry)

            # 广播到所有连接
            await broadcast_log(log_entry)
    except WebSocketDisconnect:
        active_connections.remove(websocket)
```

#### WebSocket 连接管理

```typescript
private connectWebSocket() {
  this.ws = new WebSocket('ws://localhost:38080/api/logs/stream');

  this.ws.onopen = () => {
    // 发送缓冲的日志
    while (this.buffer.length > 0) {
      const entry = this.buffer.shift();
      if (entry) {
        this.ws?.send(JSON.stringify(entry));
      }
    }
  };

  this.ws.onclose = () => {
    // 5 秒后重连
    if (this.enabled) {
      this.reconnectTimer = window.setTimeout(() => {
        this.connectWebSocket();
      }, 5000);
    }
  };
}
```

#### 启用/禁用控制

```typescript
enable() {
  this.enabled = true;
  this.connectWebSocket();
}

disable() {
  this.enabled = false;
  if (this.ws) {
    this.ws.close();
    this.ws = null;
  }
  this.buffer = [];
}
```

#### JavaScript 功能实现

```javascript
function startLogging() {
  logWs = new WebSocket('ws://localhost:38080/api/logs/stream');

  logWs.onmessage = (event) => {
    const log = JSON.parse(event.data);
    addLogEntry(log);
  };

  logWs.onopen = () => {
    // 通知主应用开始日志捕获
    if (window.opener && !window.opener.closed) {
      window.opener.postMessage({ type: 'enable-logging' }, '*');
    }

    // 加载后端日志
    loadBackendLogs();
  };
}

function addLogEntry(log) {
```

## 注意事项

1. **性能影响**
   - 日志捕获会略微增加性能开销
   - 建议仅在需要调试时启用
   - 停止监控后不会影响性能

2. **存储空间**
   - 日志文件会占用磁盘空间
   - 定期清理旧日志
   - 单个日志文件大小取决于日志量

3. **隐私安全**
   - 日志可能包含敏感信息
   - 不要将日志文件提交到 Git
   - 导出日志前检查内容

4. **WebSocket 连接**
   - 需要后端服务运行
   - 连接断开会自动重连（5 秒后）
   - 断开期间日志会缓冲（最多 100 条）
