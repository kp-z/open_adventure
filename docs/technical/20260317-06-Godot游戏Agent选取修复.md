# Godot 游戏模式 Agent 选取无响应问题修复

**创建日期**：2026-03-17
**状态**：已完成

## 问题描述

用户报告 Godot 游戏模式中无法正常选取 agent，点击 agent 无响应。

## 根本原因分析

经过代码分析，发现以下三个关键问题：

### 1. API 端口不匹配
- **Godot 客户端配置**：`http://localhost:8000/api`
- **后端实际端口**：`http://localhost:38080/api`
- **影响**：所有 API 请求都会失败（连接被拒绝）

### 2. 绑定 API 方法不匹配
- **Godot 客户端**：使用 `POST` 方法
- **后端实现**：使用 `PUT` 方法
- **影响**：即使端口正确，绑定请求也会返回 404 或 405 错误

### 3. 解绑 API 端点和方法不匹配
- **Godot 客户端**：`POST /microverse/characters/{name}/unbind`
- **后端实现**：`DELETE /microverse/characters/{name}/bind`
- **影响**：解绑功能完全无法工作

## 修复方案

修改 Godot API 客户端 `microverse/script/api/MicroverseAPIClient.gd`，使其与后端 API 保持一致。

### 修改内容

#### 1. 修正 API 端口（第 6 行）
```gdscript
# 修改前
const BASE_URL = "http://localhost:8000/api"

# 修改后
const BASE_URL = "http://localhost:38080/api"
```

#### 2. 修正绑定方法（第 36 行）
```gdscript
# 修改前
var result = await _make_request("POST", url, body)

# 修改后
var result = await _make_request("PUT", url, body)
```

#### 3. 修正解绑端点和方法（第 41-42 行）
```gdscript
# 修改前
var url = "/microverse/characters/%s/unbind" % character_name
var result = await _make_request("POST", url)

# 修改后
var url = "/microverse/characters/%s/bind" % character_name
var result = await _make_request("DELETE", url)
```

## 验证步骤

### 1. 确认服务运行状态
```bash
# 检查后端（应该在 38080 端口）
lsof -i :38080 | grep LISTEN
```

### 2. 测试 Agent 选取功能
1. 访问 `http://localhost:38080/microverse`
2. 等待游戏加载完成
3. 点击选中一个角色
4. 按 `Shift+S` 打开 Agent 配置对话框
5. 选择一个 Agent
6. 点击"绑定"按钮
7. 观察是否显示成功消息

### 3. 检查浏览器控制台
打开浏览器开发者工具（F12），查看：
- **Console 标签**：检查是否有错误
- **Network 标签**：检查 API 请求
  - 应该看到 `PUT http://localhost:38080/api/microverse/characters/{name}/bind` 返回 200
  - 如果返回 404/405，说明端点或方法仍然不匹配

### 4. 检查 Godot 游戏日志
在浏览器控制台中，查找以 `[MicroverseAPIClient]` 开头的日志：
```
[MicroverseAPIClient] PUT http://localhost:38080/api/microverse/characters/xxx/bind
[MicroverseAPIClient] Body: {"agent_id": 1}
```

## 注意事项

### 浏览器缓存问题
如果修改后游戏行为未改变，可能是浏览器缓存了旧的游戏文件：
1. 清除浏览器缓存（Ctrl+Shift+Delete）
2. 或使用隐私模式（Ctrl+Shift+N）
3. 或在 URL 后添加时间戳：`/microverse?t=123456`

### CORS 配置
确保后端 `backend/app/main.py` 中的 CORS 配置允许前端域名：
```python
app.add_middleware(
    CORSMiddleware,
    allow_origins=["http://localhost:5173"],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```
