# iOS Safari Terminal 移动端优化测试指南

**创建日期**: 2026-03-19
**测试版本**: v1.4.0+
**测试设备**: iPhone/iPad (iOS 13+)

## 测试前准备

### 1. 获取访问地址

前端开发服务器已启动，访问地址：
- **本地网络**: `http://<你的Mac IP>:5173`
- **localhost**: `http://localhost:5173` (仅限 Mac 本机)

### 2. 获取 Mac IP 地址

在 Mac 上执行：
```bash
ifconfig | grep "inet " | grep -v 127.0.0.1 | awk '{print $2}'
```

或者：
- 打开"系统设置" → "网络"
- 查看当前连接的 Wi-Fi 或以太网
- 找到 IP 地址（例如：192.168.1.100）

### 3. 确保设备在同一网络

- iPhone/iPad 和 Mac 必须连接到同一个 Wi-Fi 网络
- 关闭 Mac 的防火墙或允许端口 5173

## 测试场景

### 场景 1: visualViewport 键盘适配

**目标**: 验证键盘弹起时布局稳定，输入框不被遮挡

**步骤**:
1. 在 iPhone Safari 中打开 `http://<Mac IP>:5173`
2. 导航到 Terminal 页面
3. 创建或选择一个终端
4. 点击底部输入框

**预期结果**:
- ✅ 键盘弹起时，输入框应该贴着键盘上沿
- ✅ 输入框不被键盘遮挡
- ✅ 输出区底部有足够的 padding，内容不被遮挡
- ✅ 布局过渡平滑，无抖动

**调试信息**:
- 打开 Safari 开发者工具（Mac Safari → 开发 → iPhone → 选择页面）
- 查看控制台输出：`[useVisualViewport] Viewport changed`
- 检查 `keyboardHeight` 和 `isKeyboardVisible` 值

### 场景 2: 快捷键栏显示

**目标**: 验证快捷键栏正常显示和交互

**步骤**:
1. 点击输入框，键盘弹起
2. 点击输入框左侧的 `>` 按钮（展开快捷键）

**预期结果**:
- ✅ 快捷键栏从底部滑入
- ✅ 显示四类快捷键：
  - 紫色：`↑` `↓` (历史命令)
  - 蓝色：`|` `&&` `||` `>` `>>` `--help` (插入类)
  - 橙色：`Tab` `Ctrl+C` `Ctrl+D` `Esc` (控制类)
  - 绿色：`ls` `clear` `pwd` `cd ..` (命令类)
- ✅ 快捷键栏可以横向滚动
- ✅ 点击 `<` 按钮可以收起快捷键栏

### 场景 3: 插入类快捷键

**目标**: 验证插入类快捷键插入到光标处

**步骤**:
1. 在输入框中输入 `echo hello`
2. 将光标移动到 `hello` 之前
3. 展开快捷键栏
4. 点击 `|` 快捷键

**预期结果**:
- ✅ `|` 被插入到光标位置
- ✅ 输入框内容变为 `echo | hello`
- ✅ 光标位置在 `|` 之后
- ✅ 输入框保持聚焦状态

**其他插入类快捷键测试**:
- `&&`: 插入 ` && `
- `||`: 插入 ` || `
- `>`: 插入 ` > `
- `>>`: 插入 ` >> `
- `--help`: 插入 ` --help`

### 场景 4: 控制类快捷键

**目标**: 验证控制类快捷键发送控制事件

**步骤**:
1. 在终端中输入一些内容（不要按回车）
2. 展开快捷键栏
3. 点击 `Ctrl+C` 快捷键

**预期结果**:
- ✅ 终端显示 `^C`

**其他控制类快捷键测试**:

### 场景 5: 命令类快捷键

**目标**: 验证命令类快捷键立即执行

**步骤**:
1. 展开快捷键栏
2. 点击 `ls` 快捷键

**预期结果**:

**其他命令类快捷键测试**:

- `pwd`: 显示当前目录

### 场景 6: 历史命令导航

**目标**: 验证历史命令切换功能

**步骤**:
1. 在输入框中输入 `echo test1` 并发送
2. 输入 `echo test2` 并发送
3. 输入 `echo test3` 并发送
4. 在输入框中输入 `draft` (不发送)
5. 展开快捷键栏
6. 点击 `↑` 按钮

**预期结果**:
- ✅ 输入框显示 `echo test3` (最后一条命令)
- ✅ 再次点击 `↑`，显示 `echo test2`
- ✅ 再次点击 `↑`，显示 `echo test1`
- ✅ 点击 `↓`，显示 `echo test2`
- ✅ 继续点击 `↓` 直到恢复草稿 `draft`

**边界情况测试**:
- 在历史命令的第一条时点击 `↑`：保持不变
- 在草稿状态时点击 `↓`：保持不变
- 切换历史后修改内容再点击 `↓`：恢复原始草稿

### 场景 7: 触觉反馈

## 已知问题

### 问题 1: visualViewport 在 iOS 12 不支持

**影响**: iOS 12 及以下版本无法使用精确的键盘高度计算

**解决方案**:
- 已实现降级方案，使用 `window.innerHeight` 估算
- 功能可用，但精度略低

### 问题 2: 中文输入法 Enter 键

**影响**: 中文输入时按 Enter 可能误触发发送

**解决方案**:
- 待实施 IME 输入法处理
- 监听 `compositionstart/end` 事件
- Enter 键处理时检查 `isComposing` 状态

## 调试技巧

### 1. 启用 Safari 远程调试

**Mac 端**:
1. 打开 Safari → 偏好设置 → 高级
2. 勾选"在菜单栏中显示开发菜单"

**iPhone 端**:
1. 设置 → Safari → 高级
2. 开启"Web 检查器"

**连接调试**:
1. 用数据线连接 iPhone 和 Mac
2. 在 iPhone Safari 中打开测试页面
3. Mac Safari → 开发 → [你的 iPhone] → 选择页面
4. 打开 Web 检查器

### 2. 查看 visualViewport 日志

在控制台中查看：
```javascript
// 查看当前键盘高度
console.log('[useVisualViewport] keyboardHeight:', keyboardHeight);

// 查看 visualViewport 变化
window.visualViewport.addEventListener('resize', () => {
  console.log('visualViewport:', {
    height: window.visualViewport.height,
    offsetTop: window.visualViewport.offsetTop,
    windowHeight: window.innerHeight
  });
});
```

### 3. 启用 Terminal 调试模式

在控制台中执行：
```javascript
localStorage.setItem('terminal_debug', '1');
location.reload();
```

查看详细的终端状态日志。
