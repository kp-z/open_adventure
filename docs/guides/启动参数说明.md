# 启动参数说明

## 概述

`start.sh` 脚本支持多种启动参数，用于不同的运行场景和维护需求。

## 可用参数

### 1. `--daemon` / `-d`
**后台运行模式**

在后台运行服务，不占用终端。

```bash
./start.sh --daemon
```

**特点：**
- 前端和后端都在后台运行
- 日志输出到 `docs/logs/backend.log` 和 `docs/logs/frontend.log`
- 使用 `./stop.sh` 停止服务
- 适合服务器部署

### 2. `--clear-cache`
**清除缓存模式**

清除前端缓存和 localStorage，强制重新初始化。

```bash
./start.sh --clear-cache
```

**清除内容：**
- 前端构建缓存（`.vite/`, `dist/`, `node_modules/.vite/`）
- localStorage 所有数据（初始化标记、版本号等）

**使用场景：**
- 测试首次启动流程
- 版本升级测试
- 用户数据重置
- 开发调试

### 3. `--reset-all`
**完全重置模式**

清理所有运行时数据、依赖和配置文件。

```bash
./start.sh --reset-all
```

**清除内容：**
- 后端虚拟环境（`backend/venv/`）
- 前端依赖（`frontend/node_modules/`）
- 配置文件（`backend/.env`, `frontend/.env.local`）
- 前端缓存（包含 `--clear-cache` 的所有内容）
- 运行中的进程

**使用场景：**
- 依赖损坏需要重新安装
- 配置文件错误需要重置
- 完全清理后重新开始

### 4. `--no-sleep`
**防止休眠模式（仅 macOS）**

使用 `caffeinate` 防止系统休眠。

```bash
./start.sh --no-sleep
```

**特点：**
- 仅在 macOS 上有效
- 防止系统自动休眠
- 适合长时间运行的任务

## 参数组合

可以同时使用多个参数：

```bash
# 清除缓存 + 后台运行
./start.sh --clear-cache --daemon

# 清除缓存 + 防止休眠
./start.sh --clear-cache --no-sleep

# 完全重置 + 后台运行
./start.sh --reset-all --daemon

# 所有参数组合
./start.sh --clear-cache --daemon --no-sleep
```

## 参数对比表

| 参数 | 清除缓存 | 清除依赖 | 清除配置 | 清除进程 | 后台运行 | 防休眠 |
|------|---------|---------|---------|---------|---------|--------|
| `--clear-cache` | ✅ | ❌ | ❌ | ❌ | ❌ | ❌ |
| `--reset-all` | ✅ | ✅ | ✅ | ✅ | ❌ | ❌ |
| `--daemon` | ❌ | ❌ | ❌ | ❌ | ✅ | ❌ |
| `--no-sleep` | ❌ | ❌ | ❌ | ❌ | ❌ | ✅ |

## 常见使用场景

### 开发调试
```bash
# 测试首次启动
./start.sh --clear-cache

# 完全重置后重新开始
./start.sh --reset-all
```

### 服务器部署
```bash
# 后台运行
./start.sh --daemon

# 后台运行 + 防止休眠（macOS）
./start.sh --daemon --no-sleep
```

### 故障排查
```bash
# 清除缓存重试
./start.sh --clear-cache

# 完全重置
./start.sh --reset-all
```

### 版本升级
```bash
# 清除缓存，触发重新初始化
./start.sh --clear-cache
```

## 启动流程

### 标准启动
1. 检查插件安装
2. 设置后端环境
3. 检查端口占用
4. 启动后端服务
5. 健康检查
6. 启动前端服务

### 清除缓存启动
1. 检查插件安装
2. 设置后端环境
3. **清除前端缓存**
4. **创建清除标记文件**
5. 检查端口占用
6. 启动后端服务
7. 健康检查
8. 启动前端服务
9. **前端检测标记并清除 localStorage**

### 完全重置启动
1. **停止所有进程**
2. **删除虚拟环境和依赖**
3. **删除配置文件**
4. **清除缓存**
5. 检查插件安装
6. 重新创建虚拟环境
7. 重新安装依赖
8. 重新创建配置文件
9. 启动服务

## 注意事项

1. **数据安全**
   - `--clear-cache` 只清除前端缓存，不影响后端数据库
   - `--reset-all` 会删除配置文件，需要重新配置

2. **端口冲突**
   - 启动前会自动检查端口占用
   - 可以选择清理进程或使用新端口

3. **依赖安装**
   - `--reset-all` 会重新安装所有依赖，耗时较长
   - 首次启动会自动安装依赖

4. **日志查看**
   - 后台模式日志位于 `docs/logs/`
   - 前台模式日志直接输出到终端

## 停止服务

### 前台模式
按 `Ctrl+C` 停止

### 后台模式
```bash
./stop.sh
```

## 故障排查

### 问题：端口被占用
**解决：**
```bash
# 选项 1: 清理进程
./start.sh  # 选择 [r] 重启

# 选项 2: 使用新端口
./start.sh  # 选择 [n] 使用新端口
```

### 问题：依赖损坏
**解决：**
```bash
./start.sh --reset-all
```

### 问题：缓存导致异常
**解决：**
```bash
./start.sh --clear-cache
```

### 问题：配置文件错误
**解决：**
```bash
./start.sh --reset-all
# 然后编辑 backend/.env 重新配置
```
