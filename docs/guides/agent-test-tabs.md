# AgentTestPanel Tab 切换功能

## 功能说明

使用 Tab 切换显示"输出结果"和"控制台日志"，让界面更清晰。

## UI 布局

### 运行时
```
┌─────────────────────────────────────┐
│ 测试控制台                           │
├─────────────────────────────────────┤
│ [输入区域]                           │
├─────────────────────────────────────┤
│ 🔄 Agent 正在处理请求...            │
│                                     │
│ [00:48:19] 开始执行 Agent: xxx      │
│ [00:48:19] 使用模型: sonnet         │
│ [00:48:20] 正在分析输入...          │
│ ↓ (自动滚动)                        │
└─────────────────────────────────────┘
```

### 完成后
```
┌─────────────────────────────────────┐
│ 测试控制台                           │
├─────────────────────────────────────┤
│ [输入区域]                           │
├─────────────────────────────────────┤
│ [输出结果] [控制台日志]              │
├─────────────────────────────────────┤
│ ✅ 执行成功 | 1.23s · sonnet  [复制]│
│                                     │
│ 我是一个专门的 Agent...             │
└─────────────────────────────────────┘
```

## 交互逻辑

### 1. 运行时
- 自动切换到"控制台日志" Tab
- 实时显示日志输出
- 自动滚动到最新

### 2. 完成时
- 自动切换到"输出结果" Tab
- 显示最终输出
- 控制台日志保存到 localStorage

### 3. Tab 切换
- 点击"输出结果"：显示最终输出（支持 Markdown）
- 点击"控制台日志"：显示执行过程的日志

## 数据存储

```typescript
// 运行时
consoleLogs[] -> 实时显示

// 完成时
localStorage.setItem(`agent-test-logs-${agent.id}`, consoleLogs.join('\n'))
currentOutput = finalOutput

// Tab 切换
activeTab === 'output' -> 显示 currentOutput
activeTab === 'logs' -> 显示 localStorage 中的日志
```

## 优势

1. **清晰分离**
   - 输出结果：最终答案
   - 控制台日志：执行过程

2. **自动切换**
   - 运行时看日志
   - 完成后看结果

3. **保留历史**
   - 日志保存在 localStorage
   - 随时可以切换查看
