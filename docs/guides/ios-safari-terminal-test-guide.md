# iOS Safari Terminal 移动端优化测试指南

**创建日期**: 2026-03-19
**测试版本**: v1.4.0+
**测试环境**: iOS Safari

## 测试准备

### 1. 访问测试环境

**本地开发环境**:
```
http://localhost:5173
```

**通过局域网访问**（推荐）:
1. 获取 Mac 的局域网 IP 地址：
   ```bash
   ifconfig | grep "inet " | grep -v 127.0.0.1
   ```
2. 在 iPhone/iPad Safari 中访问：
   ```
   http://<Mac-IP>:5173
   ```
   例如：`http://192.168.1.100:5173`

### 2. 测试设备要求

- **iOS 版本**: iOS 13+ (支持 visualViewport API)
- **浏览器**: Safari (系统自带)
- **推荐设备**:
  - iPhone 13 Pro 或更新
  - iPad Air 或更新

### 3. 开启调试模式（可选）

在 Safari 中打开控制台查看日志：
1. iPhone 设置 → Safari → 高级 → 网页检查器（开启）
2. Mac Safari → 开发 → [你的 iPhone] → localhost

## 测试场景

### 场景 1: visualViewport 键盘适配

**目标**: 验证键盘弹起时布局稳定，输入框不被遮挡

**步骤**:
1. 打开 Terminal 页面
2. 点击底部输入框
3. 观察键盘弹起时的布局变化

**预期结果**:
- ✅ 输入栏紧贴键盘上沿
- ✅ 输出区底部不被键盘遮挡
- ✅ 输入栏有足够的 padding，内容不被遮挡
- ✅ 键盘收起时布局恢复正常

**检查点**:
- 输入栏是否跟随键盘移动？
- 终端输出区是否有足够的底部 padding？
- 键盘弹起/收起时是否有抖动？

**调试信息**（开发模式）:
打开浏览器控制台，查看日志：
```
[useVisualViewport] Viewport changed: {
  windowHeight: 844,
  vvHeight: 500,
  vvOffsetTop: 0,
  keyboardHeight: 344,
  isVisible: true
}
```

### 场景 2: 快捷键栏功能

**目标**: 验证快捷键栏显示和交互

**步骤**:
1. 点击输入框，键盘弹起
2. 点击输入框左侧的 `>` 按钮（展开快捷键）
3. 观察快捷键栏显示

**预期结果**:
- ✅ 快捷键栏在输入框上方显示
- ✅ 横向滚动流畅
- ✅ 四种颜色区分不同类型快捷键：
  - 紫色：历史命令（↑ ↓）
  - 蓝色：插入类（| && || > >> --help）
  - 橙色：控制类（Tab Ctrl+C Ctrl+D Esc）
  - 绿色：命令类（ls clear pwd cd ..）

**检查点**:
- 快捷键栏是否完整显示？
- 横向滚动是否流畅？
- 颜色区分是否清晰？

### 场景 3: 历史命令导航

**目标**: 验证历史命令切换功能

**步骤**:
1. 在输入框中输入几条命令并执行：
   - `ls`
   - `pwd`
   - `echo "test"`
2. 展开快捷键栏
3. 点击 `↑` 按钮
4. 观察输入框内容变化
5. 继续点击 `↑` 查看更早的命令
6. 点击 `↓` 返回较新的命令

**预期结果**:
- ✅ `↑` 切换到上一条历史命令
- ✅ `↓` 切换到下一条历史命令
- ✅ 到达历史顶部时，`↑` 不再变化
- ✅ 到达历史底部时，恢复当前草稿
- ✅ 有触觉反馈（震动）

**检查点**:
- 历史命令顺序是否正确？
- 草稿是否正确保存和恢复？
- 是否有触觉反馈？

### 场景 4: 插入类快捷键

**目标**: 验证插入类快捷键插入到光标处

**步骤**:
1. 在输入框中输入：`ls`
2. 将光标移到 `ls` 之后
3. 点击快捷键 `|`
4. 观察输入框内容

**预期结果**:
- ✅ 输入框内容变为：`ls | `
- ✅ 光标在 `|` 之后
- ✅ 可以继续输入

**测试其他插入类快捷键**:
- `&&` → 插入 ` && `
- `||` → 插入 ` || `
- `>` → 插入 ` > `
- `>>` → 插入 ` >> `
- `--help` → 插入 ` --help`

**检查点**:
- 是否插入到光标位置？
- 插入后光标位置是否正确？
- 是否有空格包围？

### 场景 5: 控制类快捷键

**目标**: 验证控制类快捷键发送控制事件

**步骤**:
1. 点击 `Tab` 快捷键
2. 观察终端是否收到 Tab 键
3. 点击 `Ctrl+C` 快捷键
4. 观察终端是否收到中断信号

**预期结果**:
- ✅ `Tab` 发送 Tab 键到终端
- ✅ `Ctrl+C` 发送中断信号
- ✅ `Ctrl+D` 发送 EOF 信号
- ✅ `Esc` 发送 Escape 键

**检查点**:
- 控制键是否正确发送到终端？
- 终端是否正确响应？

### 场景 6: 命令类快捷键

**目标**: 验证命令类快捷键立即执行

**步骤**:
1. 点击 `ls` 快捷键
2. 观察终端输出

**预期结果**:
- ✅ 输入框显示 `ls`
- ✅ 命令自动执行
- ✅ 终端显示 `ls` 输出
- ✅ 输入框清空

**测试其他命令类快捷键**:
- `clear` → 清屏
- `pwd` → 显示当前目录
- `cd ..` → 返回上级目录

**检查点**:
- 命令是否自动执行？
- 输入框是否清空？
- 终端输出是否正确？

### 场景 7: 键盘收起后状态

**目标**: 验证键盘收起后布局恢复

**步骤**:
1. 点击输入框，键盘弹起
2. 点击终端输出区域
3. 键盘收起

**预期结果**:
- ✅ 输入栏移回底部
- ✅ 快捷键栏隐藏
- ✅ 终端输出区恢复正常高度
- ✅ 底部 dock 重新显示

**检查点**:
- 布局是否完全恢复？
- 是否有残留的偏移？

### 场景 8: 横屏测试

**目标**: 验证横屏模式下的表现

**步骤**:

**预期结果**:

## 已知问题和限制

### 1. iOS 版本兼容性

**问题**: iOS 13 以下不支持 `visualViewport` API

**影响**: 键盘高度计算可能不准确

**降级方案**: 使用 `window.innerHeight` 估算

**验证方法**:
- 在 iOS 12 设备上测试
- 检查是否有降级提示

### 2. 中文输入法

**问题**: 中文输入时 Enter 键行为

**影响**: 可能误触发命令执行

**解决方案**: 添加 IME 输入法检测（待实施）

**验证方法**:
- 使用中文输入法输入
- 按 Enter 确认输入
- 检查是否误执行命令

### 3. 性能问题

**问题**: `visualViewport` 事件频繁触发

**影响**: 可能导致性能问题

**优化方案**: 使用 `requestAnimationFrame` 节流

**验证方法**:
- 快速滑动键盘
- 检查是否有卡顿
- 查看控制台日志频率

## 调试技巧

### 1. 查看 visualViewport 日志

在 Safari 控制台中运行：
```javascript
window.visualViewport.addEventListener('resize', () => {
  console.log({
    height: window.visualViewport.height,
    offsetTop: window.visualViewport.offsetTop,
    keyboardHeight: window.innerHeight - window.visualViewport.height - window.visualViewport.offsetTop
  });
});
```

### 2. 检查快捷键事件

在控制台中运行：
```javascript
localStorage.setItem('terminal_debug', '1');
location.reload();
```

### 3. 模拟键盘高度

在控制台中运行：
```javascript
// 模拟键盘弹起
document.documentElement.style.setProperty('--keyboard-height', '344px');

// 模拟键盘收起
document.documentElement.style.setProperty('--keyboard-height', '0px');
```
