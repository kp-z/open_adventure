# 模型配置指南

**创建日期**: 2026-03-10
**状态**: 已发布

## 概述

Open Adventure 支持灵活的模型配置，允许用户自定义 Claude 模型列表。系统采用混合方案：
- 内置默认模型列表（开箱即用）
- 支持用户自定义模型（可扩展）
- 自动去重，用户配置优先

## 默认模型列表

系统内置以下 Claude 模型：

| 别名 | 完整名称 | 描述 |
|------|----------|------|
| haiku | claude-haiku-4-6 | Fast and efficient |
| sonnet | claude-sonnet-4-6 | Balanced performance |
| opus | claude-opus-4-6 | Most capable model |
| opus[1m] | claude-opus-4-6[1m] | Opus with 1M context |
| haiku-3.5 | claude-3-5-haiku-20241022 | Haiku 3.5 |
| sonnet-3.5 | claude-3-5-sonnet-20241022 | Sonnet 3.5 |
| sonnet-4 | claude-sonnet-4-20250514 | Sonnet 4 |
| opus-4 | claude-opus-4-20250514 | Opus 4 |
| opus-4.5 | claude-opus-4-5-20250514 | Opus 4.5 |

## 自定义模型配置

### 配置文件位置

用户自定义模型配置文件：`~/.claude/custom_models.json`

### 配置文件格式

```json
{
  "models": [
    {
      "alias": "my-custom-model",
      "full_name": "claude-custom-20260101",
      "description": "My custom model"
    }
  ]
}
```

### 字段说明

- `alias`: 模型别名，用于在 UI 中显示和选择
- `full_name`: 模型的完整 API 名称
- `description`: 模型描述，显示在 UI 中

### 使用示例

#### 1. 添加新模型

```json
{
  "models": [
    {
      "alias": "opus-5",
      "full_name": "claude-opus-5-20260301",
      "description": "Latest Opus 5 model"
    }
  ]
}
```

#### 2. 覆盖默认模型

如果自定义模型的 `alias` 与默认模型相同，将覆盖默认配置：

```json
{
  "models": [
    {
      "alias": "opus",
      "full_name": "claude-opus-custom",
      "description": "Custom opus configuration"
    }
  ]
}
```

#### 3. 添加多个模型

```json
{
  "models": [
    {
      "alias": "opus-5",
      "full_name": "claude-opus-5-20260301",
      "description": "Latest Opus 5 model"
    },
    {
      "alias": "sonnet-5",
      "full_name": "claude-sonnet-5-20260301",
      "description": "Latest Sonnet 5 model"
    }
  ]
}
```

## 模型可用性检测

系统会自动检测每个模型的可用性：
- ✅ 绿色气泡：模型可用
- ⚪ 灰色气泡：模型不可用或无权限

检测逻辑：
1. 读取 `~/.claude/settings.json` 中的 API key
2. 向 Anthropic API 发送测试请求（1 token）
3. 根据响应状态判断可用性

## 环境适配性

### 场景 1: 只有 sonnet 可用的环境

**表现**: ✅ 正常工作
- 系统显示所有模型
- sonnet 显示为可用（绿色）
- 其他模型显示为不可用（灰色）

### 场景 2: 新模型发布

**解决方案**: 添加自定义模型配置

```bash
# 创建配置文件
cat > ~/.claude/custom_models.json << 'EOF'
{
  "models": [
    {
      "alias": "opus-5",
      "full_name": "claude-opus-5-20260301",
      "description": "Latest Opus 5 model"
    }
  ]
}
EOF

# 重启后端
cd /path/to/open_adventure
./start.sh
```

### 场景 3: 迁移到不同环境

**步骤**:
1. 复制项目到新环境
2. 配置 API key（`~/.claude/settings.json`）
3. （可选）创建自定义模型配置
4. 启动服务

系统会自动检测可用模型，无需修改代码。

## 配置文件位置

| 文件 | 位置 | 用途 |
|------|------|------|
| 默认模型配置 | `backend/app/config/default_models.json` | 系统内置模型列表 |
| 自定义模型配置 | `~/.claude/custom_models.json` | 用户自定义模型 |
| 配置示例 | `backend/app/config/custom_models.example.json` | 自定义配置示例 |

## 验证配置

### 1. 检查模型数量

```bash
curl -s http://localhost:38080/api/claude/health | jq '.model_info.available_models | length'
```

### 2. 查看所有模型

```bash
curl -s http://localhost:38080/api/claude/health | jq '.model_info.available_models | map({alias, description, available})'
```

### 3. 检查特定模型

```bash
curl -s http://localhost:38080/api/claude/health | jq '.model_info.available_models[] | select(.alias == "opus")'
```

## 故障排查

### 问题 1: 自定义模型未显示

**原因**: 配置文件格式错误或位置不正确

**解决方案**:
1. 检查文件位置：`~/.claude/custom_models.json`
2. 验证 JSON 格式：`cat ~/.claude/custom_models.json | jq .`
3. 检查后端日志：查看是否有加载错误

### 问题 2: 所有模型显示为不可用

**原因**: API key 未配置或无效

**解决方案**:
1. 检查 `~/.claude/settings.json` 中的 `ANTHROPIC_AUTH_TOKEN`
2. 验证 API key 是否有效
3. 检查网络连接

### 问题 3: 自定义模型未覆盖默认模型

**原因**: `alias` 不匹配

**解决方案**:
确保自定义模型的 `alias` 与默认模型完全一致（区分大小写）

## 最佳实践

1. **备份配置**: 修改前备份 `custom_models.json`
2. **验证格式**: 使用 `jq` 验证 JSON 格式
3. **测试可用性**: 添加新模型后，检查 Dashboard 确认可用性
4. **文档记录**: 在团队中共享自定义模型配置
